# Common Bug Fix Patterns

> **NEVER modify generated files directly.** All files marked `// Code generated by ack-generate. DO NOT EDIT.` (including `sdk.go`, `delta.go`, `manager.go`, `resource.go`, etc.) will be overwritten on the next code generation run. Fix bugs by changing `generator.yaml`, hook templates (`.go.tpl`), or `hooks.go` — then regenerate.

Reference for the most frequent root causes found in closed ACK bugs. When investigating a new bug, match the symptom to a pattern below for the likely fix approach.

## Pattern 1: Infinite Reconcile Due to JSON/Whitespace/Ordering Differences

**Symptoms:** Controller constantly reconciles detecting diffs on the same field. Diff shows the same semantic value with different formatting (whitespace, key ordering, redundant fields). Common with IAM policy documents, JSON strings, and list fields.

**Root Cause:** AWS API returns a normalized/reformatted version of a field that differs textually but is semantically identical.

**Fix:**
- **IAM policy fields:** Add `is_iam_policy: true` in `generator.yaml` → triggers `IAMPolicyDocumentEqual` comparator
- **Unordered lists:** `compare.is_ignored: true` + `delta_pre_compare` hook with custom normalization
- **Server-added defaults:** `late_initialize: {}` on those fields

**Example (sns-controller PR #111):**
```yaml
Policy:
  is_iam_policy: true
```

## Pattern 2: Field Ignored in Delta but Missing Custom Compare Hook

**Symptoms:** User changes a field value, but no update is propagated. Controller logs show no delta detected. Resource stays synced.

**Root Cause:** Field is `compare.is_ignored: true` but the corresponding manual comparison in `delta_pre_compare` hook was never added or has a typo.

**Fix:**
1. Find `customPreCompare` in `hooks.go` or template
2. Add the missing field comparison using `ackcompare.HasNilDifference` + value equality

**Example:**
```go
if ackcompare.HasNilDifference(a.ko.Spec.AllocatedStorage, b.ko.Spec.AllocatedStorage) {
    delta.Add("Spec.AllocatedStorage", a.ko.Spec.AllocatedStorage, b.ko.Spec.AllocatedStorage)
} else if a.ko.Spec.AllocatedStorage != nil && b.ko.Spec.AllocatedStorage != nil {
    if *a.ko.Spec.AllocatedStorage != *b.ko.Spec.AllocatedStorage {
        delta.Add("Spec.AllocatedStorage", a.ko.Spec.AllocatedStorage, b.ko.Spec.AllocatedStorage)
    }
}
```

## Pattern 3: Unchanged Field Sent in Update Causes API Error

**Symptoms:** Modifying any field triggers a terminal error about a *different* field. Error like "Modification of X is not supported" even though X wasn't changed.

**Root Cause:** Generated `sdkUpdate` builds input from all spec fields. Some AWS APIs reject update calls if certain fields are included even when unchanged.

**Fix:** Add `sdk_update_post_build_request` hook to nil out the field when not in delta:
```go
if !delta.DifferentAt("Spec.FieldName") {
    input.FieldName = nil
}
```
Or mark with `is_immutable: true` if the field truly cannot change after creation.

## Pattern 4: Nil Pointer Panic in Hook Code

**Symptoms:** Controller panics with `invalid memory address or nil pointer dereference`. Resource created in AWS but status never updates. Triggered by optional fields that are nil in certain states.

**Root Cause:** Custom hook dereferences a pointer without nil-check. AWS responses have nested optional structs where inner fields are nil when a feature is disabled.

**Fix:** Add nil checks before every pointer dereference in the chain. Convert invalid input to a terminal error rather than panicking.

## Pattern 5: ReadMany Returns Wrong Resource (Missing Filter)

**Symptoms:** Creating a resource gives "Resource already exists". Multiple CRs point to the same AWS resource. Controller adopts unrelated resources.

**Root Cause:** Generated `sdkFind` uses a List API without filtering by the specific resource's identifier.

**Fix:** Add `list_operation.match_fields` in `generator.yaml`:
```yaml
list_operation:
  match_fields:
    - ServiceId
```
Or add `sdk_read_many_post_build_request` hook to set filter parameters.

## Pattern 6: Tags Not Updating After Creation

**Symptoms:** Tags set correctly on creation but updates to tags in CR spec have no effect.

**Root Cause:** Missing/broken `syncTags` hook, or tag update API requires separate calls (AddTags/RemoveTags) not wired in update path. Common bug: ARN and tag list parameters swapped.

**Fix:** Implement or fix `ensure_tags` hook that calls the correct tag API with correct parameter order. Verify the tag diff logic computes add/remove sets correctly.

## Pattern 7: Adoption Fails with "Resource Not Found"

**Symptoms:** Resource adoption via annotation fails. "Adopted resource not found" or fields not populated after adoption.

**Root Cause:** Resource's primary identifier for ReadOne doesn't match what adoption provides. Or `sdkFind` checks a status field that was never set during adoption.

**Fix:**
- Set `is_arn_primary_key: true` if identified by ARN
- Guard status-field checks with nil: `if ko.Status.ID != nil { ... }`
- Verify adoption populates all fields needed for ReadOne

## Pattern 8: Deletion Succeeds in K8s but AWS Resource Persists

**Symptoms:** CR deleted (finalizer removed) but AWS resource still exists. Batch delete APIs return 200 with errors in response body.

**Root Cause:** Some AWS delete APIs return success HTTP 200 with an `Unsuccessful` array that the generated handler doesn't inspect.

**Fix:** Add `sdk_delete_post_request` hook that checks response for error entries and returns an error to prevent finalizer removal.

## Pattern 9: Immutable Field Change Creates Orphaned Resource

**Symptoms:** Changing a primary identifier field (e.g., `spec.name`) creates a new AWS resource. Old resource becomes orphaned.

**Root Cause:** Identifier field not marked immutable. Controller treats it as new creation.

**Fix:** Mark identifier fields with `is_immutable: true` in `generator.yaml`.

## Pattern 10: Custom Update Completely Non-Functional

**Symptoms:** Controller logs "updated resource" but AWS state never changes. `custom_method_name` specified but implementation has logic errors.

**Root Cause:** Custom update implementations use `diff.Path.Contains()` incorrectly (prefix matching, not substring) or compute add/remove sets wrong.

**Fix:** Replace broken `custom_method_name` with generated update + hooks (`sdk_update_pre_build_request`/`sdk_update_post_build_request`). Add `synced.when` for async resources.
